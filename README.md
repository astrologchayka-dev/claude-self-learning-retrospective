# Claude Self-Learning Retrospective

**Ретро-скилл + ресёрч, которые превращают опыт каждой сессии Claude Code в надёжные механизмы — чтобы агент становился умнее с каждой сессией, а не начинал с нуля.**

> A retrospective/self-improvement skill for Claude Code. It turns each session's lessons (failures *and* wins) into durable artifacts — hooks, skills, gates, checklists — wired into the inevitable chokepoints a future session must pass through. Content is in Russian; the method is language-agnostic. MIT-licensed.

---

## Зачем это

LLM-сессия без памяти проходит только через фиксированные точки: системный промпт, авто-загружаемый CLAUDE.md, шаблоны запуска, скиллы (по описанию), хуки и индекс памяти (первые ~200 строк). **Урок вне этих точек в следующей сессии не будет прочитан.** Значит «записать вывод в заметку» ≈ гарантировать рецидив.

Этот скилл заставляет поднимать каждый урок с уровня прозы до уровня **механизма**, который сработает неминуемо, и не считать урок закрытым, пока не доказано, что поведение реально изменилось.

## Что внутри

- **`skill/retrospective/`** — сам скилл (`SKILL.md` + `reference/mechanics-and-basis.md`). Готов к установке в Claude Code.
- **`research/self-learning-loop-for-claude.md`** — полный ресёрч-отчёт: доказательная база из 4 областей (самообучающиеся агенты, инженерия постмортемов, lean SDCA→PDCA/hansei, механика Claude Code) со всеми источниками.

## Ядро метода (TL;DR)

**Главный закон:** урок надёжен ровно настолько, насколько неминуема точка прохождения, где он лежит, и доказано, что он сменил поведение.

**Петля самообучения (8 шагов):** Захват → Дистилляция → Гейт-Revise → Маршрутизация → Индексация → Верификация смены поведения → Стандартизация (SDCA) → Check + тираж.

**Карта маршрутизации — куда класть урок (по убыванию надёжности):**

| Тип урока | Куда | Почему |
|---|---|---|
| «Всегда/никогда», блок «готово», блок опасного | **Хук** | Исполняет harness детерминированно — единственное настоящее принуждение |
| Метод «как делать X правильно» | **Скилл** (+ дубль в шаблон/DoD, если обязателен) | Авто-триггер по `description` |
| Стабильный факт / конвенция | **CLAUDE.md** | Авто-грузится, но advisory — держи < 200 строк |
| Привязан к запуску / сдаче | **Шаблон запуска / DoD-чеклист** | Процедурная точка входа |
| Надо враждебно перепроверять | **Критик-субагент** в связке с гейтом | Чистый контекст, не предвзят |
| «Просто на память» | MEMORY-индекс (первые 200 строк) | Низшая ступень — только как дубль к механизму |

Полная таблица, механика хуков/памяти и доказательная база — в [research/](research/self-learning-loop-for-claude.md) и [reference/](skill/retrospective/reference/mechanics-and-basis.md).

## Установка

```bash
git clone https://github.com/astrologchayka-dev/claude-self-learning-retrospective.git
cp -r claude-self-learning-retrospective/skill/retrospective ~/.claude/skills/
```

Или скриптом:

```bash
./install.sh
```

После установки Claude Code авто-подхватит скилл по описанию — в конце сессии, после инцидента/упущения, при «как сделать, чтобы не повторилось», при передаче между сессиями. Можно вызвать явно: `/retrospective`.

## Замечания

- Скилл написан по-русски (методология языково-независима — при желании переводится).
- Несколько ссылок вида `[[имя]]` в тексте скилла ведут в граф памяти автора и опциональны — на работу метода не влияют.
- Источник правды по механике Claude Code — официальные доки (`code.claude.com/docs`), не этот репозиторий; проверяй актуальность.
